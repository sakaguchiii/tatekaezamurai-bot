## 🔍 よくある質問

### Q1. 「どこから読めばいいかわからない」

**A**: まず **クイックスタートガイド_初心者向け.md** を読んでください。5分で全体像がつかめます。

---

### Q2. 「エラーが出たけど、どうすればいい？」

**A**: **トラブルシューティングガイド.md** を開いて、エラーメッセージを検索（Ctrl+F）してください。

---

### Q3. 「新しい機能を追加したいけど、どうすればいい？」

**A**:
1. **ユーザビリティ改善提案書.md** で提案済みの機能を確認
2. **README_開発者向け.md** でコード構造を理解
3. 新しいブランチを作成して開発開始

---

### Q4. 「このドキュメント、情報が古くない？」

**A**: 各ドキュメントの冒頭に「最終更新日」が記載されています。古い場合は、最新情報をGitHubで確認してください。

---

### Q5. 「ドキュメントを更新したい」

**A**: 大歓迎です！
```bash
# 1. ドキュメントを編集
nano クイックスタートガイド_初心者向け.md

# 2. 最終更新日を変更

# 3. コミット
git add .
git commit -m "docs: クイックスタートガイドを更新"
git push origin main
```

## 🎯 ドキュメント作成のポリシー

このプロジェクトでは、以下のポリシーでドキュメントを管理しています：

1. **初心者ファースト**
   - プログラミング初心者でも理解できる言葉を使う
   - 専門用語は必ず説明する

2. **実用性重視**
   - コピペで使えるコマンドを提供
   - チェックリストで作業を明確に

3. **更新を続ける**
   - 新しい問題が起きたら、ドキュメントを更新
   - 次の人のために残す

4. **検索しやすく**
   - エラーメッセージをそのまま記載
   - 目次を充実させる

## 📞 困ったときの連絡先

1. **まずドキュメントを検索**
   ```bash
   grep -r "キーワード" *.md
   ```

2. **GitHubのIssueを検索**
   https://github.com/sakaguchiii/tatekaezamurai-bot/issues

3. **新しいIssueを作成**
   - エラーログを添付
   - 何をしたか、何が起きたかを記載
